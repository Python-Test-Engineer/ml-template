### Change 1 — `.claude/commands/rsi.md`

**Rationale:** Expanded skill prompts (e.g., the full markdown text of `/show-convo` or `/rsi` injected as user messages) and `<command-message>` tags pass through the Step 3 noise filter, creating false "user" turns in parsed session data. This degrades signal detection accuracy.
**Type:** Addition
**Before:**
```python
                if len(stripped) < 5:
                    continue

            # Truncate to 600 chars
```
**After:**
```python
                if len(stripped) < 5:
                    continue
                if '<command-message>' in stripped:
                    continue
                if len(stripped) > 200 and stripped.startswith('# ') and '## ' in stripped:
                    continue

            # Truncate to 600 chars
```
**Signal source:** Session d1b66315 — parsed turns included `"# Show Conversations\r\n\r\n## ACTION\r\n\r\nRun..."` as a user message. Session fda6fce0 — same pattern with both `/show-convo` and `/rsi` skill expansions captured as user turns.
